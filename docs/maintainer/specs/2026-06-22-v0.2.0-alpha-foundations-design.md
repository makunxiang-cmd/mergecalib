# v0.2.0 Alpha Foundations Design

## Status

Approved design direction for the first v0.2.0 development milestone.

This milestone is an internal foundation release. It does not target CRAN
submission and does not need to preserve v0.1.0 error-message compatibility.
It must preserve the statistical and operational invariants listed below.

## Goals

1. Replace text-only error and warning handling with stable classed
   conditions.
2. Remove full-dataset distance-matrix allocation from candidate generation by
   computing distances province block by province block.
3. Add a first warning layer for valid but risky fitted results.
4. Expand tests so they assert behavior, condition classes, and invariants
   rather than brittle message text.

## Non-Goals

This milestone does not implement:

- elastic relaxation diagnostics (`relaxation = "minimax"` or `"l1"`);
- province decomposition or parallel solving;
- weight diagnostics, calibration reports, or standard errors;
- pluggable solver backends;
- roxygen migration;
- CRAN release hardening beyond keeping checks passing.

## Hard Invariants

All implementation work must preserve:

1. Final clusters have strictly positive sample size.
2. Zero-sample cells are absorbed within province only.
3. No fabricated sample and no default cross-province borrowing.
4. Deterministic output under identical inputs and package version.
5. Conservation of sample size, total weight, and per-grade counts.
6. English user-facing package messages and ASCII-only R source.

## Architecture

### Condition Module

Create `r-package/R/conditions.R` as the single home for package condition
constructors and emitters.

Public-facing exported functions do not expose these helpers directly. Internal
code uses them through:

- `.mc_error(message, class, ..., call = NULL)`
- `.mc_stop(class, ..., call = NULL)`
- `.mc_warning(message, class, ..., call = NULL)`
- `.mc_warn(class, ..., call = NULL)`

`utils.R` keeps generic helpers such as integer checks, wildcard matching,
ordering, and compact number formatting. Existing `.mc_stop()` and `.mc_warn()`
compatibility wrappers are replaced or moved so new code consistently uses
classed conditions.

### Error Classes

The first class taxonomy is:

- `mergecalib_error_input`
- `mergecalib_error_spec`
- `mergecalib_error_target`
- `mergecalib_error_structural`
- `mergecalib_error_not_estimable`
- `mergecalib_error_candidate`
- `mergecalib_error_weight_bounds`
- `mergecalib_error_solver`
- `mergecalib_error_infeasible`
- `mergecalib_error_internal`

Each condition also inherits from `mergecalib_error` and `error`.

Tests must use `expect_error(..., class = "...")`. Message checks are allowed
only for dynamic facts such as offending column names, target identifiers, or
province names.

### Warning Classes

The first warning taxonomy is:

- `mergecalib_warning_relaxation`
- `mergecalib_warning_weight_distortion`
- `mergecalib_warning_heterogeneity`
- `mergecalib_warning_near_binding`

Each condition also inherits from `mergecalib_warning` and `warning`.

Warnings are controlled by:

- `options(mergecalib.warn = TRUE/FALSE)`
- `options(mergecalib.warning_thresholds = list(...))`

Default thresholds must be conservative so the package example does not emit
warnings under normal v0.1.0-style usage.

## Candidate Distance Blocking

Current behavior allocates a dense full-dataset distance matrix before looping
through provinces. The new implementation computes distances only within the
current province block.

Implementation shape:

1. Replace `.distance_matrix(data, spec, distance_weights)` with a block helper
   such as `.distance_matrix_block(data_block, spec, distance_weights)`.
2. In `generate_candidate_clusters()`, enter the province loop before computing
   distances.
3. Use local block row numbers for distance lookups.
4. Convert local block members back to original global row numbers before
   storing candidates.
5. Keep `cluster_key`, `members`, sorting, and candidate metrics deterministic.

Acceptance criteria:

- Example candidate output keeps the same selected `cluster_key`, `members`,
  and metric values as the pre-blocking implementation.
- Candidate generation memory scales with the largest province block, not the
  full national row count.
- All zero-sample absorption behavior remains within province and deterministic.

## Warning Emission Points

Warnings are emitted after a fit is available and final outputs can be audited.
The initial warning layer should be centralized in a helper such as
`.emit_fit_warnings(fit)`.

Suggested threshold defaults:

- `relaxation_delta = 0.02`
- `max_weight_ratio = 5`
- `heterogeneity = Inf`
- `near_binding_margin = 0.005`

`heterogeneity = Inf` keeps that warning opt-in until a better empirical default
is established.

Warning suppression rules:

- `options(mergecalib.warn = FALSE)` suppresses all mergecalib warnings.
- Specific threshold values set to `Inf` suppress that risk check.
- Existing R warning controls such as `suppressWarnings()` continue to work.

## Testing Strategy

Use test-driven development for every behavior change:

1. Write the failing test.
2. Run the targeted test and verify the expected failure.
3. Implement the smallest code change that passes.
4. Run the targeted test.
5. Run the full package tests.

New or expanded tests:

- `test-conditions.R`
  - constructors produce the expected class hierarchy;
  - validation failures are catchable by specific classes;
  - solver/candidate/internal helpers use classed errors.
- `test-distance-blocking.R`
  - blocked candidate generation matches a fixture from the current algorithm;
  - zero-sample absorption remains covered and positive;
  - multi-province data never requires cross-province distances.
- `test-warnings.R`
  - relaxation warnings fire when `delta` exceeds the configured threshold;
  - weight-ratio warnings fire with an intentionally low threshold;
  - warnings are suppressed by `options(mergecalib.warn = FALSE)`.
- Existing validation/candidate/fit tests
  - migrate from message-only assertions to class assertions;
  - continue checking positive final sample size and audit validity.

## Documentation Updates

Update these files during implementation:

- `docs/maintainer/PROJECT_OVERVIEW.md`
- `docs/maintainer/ROADMAP_v0.2.0.md`
- `r-package/NEWS.md`
- relevant `r-package/man/*.Rd` files if arguments, return values, or exported
  user-visible behavior changes.

Do not add generated PDFs, built tarballs, local check directories, private
notes, or local handoff files to the repository.

## Rollout

The milestone is complete when:

1. package tests pass;
2. `R CMD build .` succeeds from `r-package/`;
3. `R CMD check --as-cran --no-manual` completes with no errors or warnings;
4. remaining notes are understood and documented;
5. maintainer docs reflect the new condition, warning, and distance-blocking
   design.

## Open Decisions

No open product decisions remain for this alpha scope. Threshold defaults may be
revised during implementation if tests show the conservative defaults are too
noisy or too weak.
